UAT Instability Pattern

5 distinct failure types emerged from a single root cause — UAT environment instability. Each deadline push revealed a new surface area of breakage: first data creation pipelines, then submission workflows, then UI tab loading. The variety of blockers is not a sign of multiple problems — it is a sign of one deeply unstable environment affecting different layers.

Resource Constraint (Apr 10 → 11)

A front-end fix (Bootstrap removal) that should have taken 2–3 hours consumed a full day because it had to be validated across 4 production environments with insufficient QA bandwidth. This is a force-multiplier risk: any change becomes 4× more expensive to test as the environment count grows.
